## рҹ“ҳ Question Summaries

### Q1 вҖ” Smart Missing Value Handling (`q1_missing_values.py`)

| Step | Detail |
|------|--------|
| Dataset | `retail_sales.csv` вҖ” 200 rows, columns: OrderID, CustomerID, ProductCategory, SalesAmount, Region, OrderDate |
| Problem | `SalesAmount` is missing only for **Electronics** in **South** region |
| Strategy 1 | Fill with **Regional Median** (same Category + Region) |
| Strategy 2 | Fallback to **Category Median** if regional group is too small |
| Audit column | `Imputation_Method` records how each row was filled |
| Output | `outputs/cleaned_retail_sales.csv` |

**Sample output:**
```
Regional_Median  : 7 rows
Category_Median  : 0 rows
```

### Q2 вҖ” Complex String Manipulation (`q2_string_manipulation.py`)

| Step | Detail |
|------|--------|
| Dataset | `employee_master.csv` вҖ” 60 rows, mixed-format names |
| Titles removed | Dr., Mr., Ms., Mrs., Prof. |
| Columns added | `First_Name`, `Middle_Initial`, `Last_Name`, `Username` |
| Username format | `firstinitial` + `lastname` (lowercase, alphanumeric only) |

### Q3 вҖ” Memory-Efficient Log Processing (`q3_memory_efficient.py`)

| Step | Detail |
|------|--------|
| Dataset | `server_logs.csv` вҖ” 5,000 rows (scales to 1.5 M+ with same code) |
| Columns used | Timestamp, UserID, Action, ResponseTime |
| Filter | `Action == "ERROR"` only |
| Aggregation | Average `ResponseTime` per `UserID` |
| Output | Top 7 users with highest average response time |

**Memory optimisation strategies used:**
- `chunksize=N` вҶ’ only N rows in RAM at any time
- `usecols=[...]` вҶ’ skip unused columns at parse time
- `dtype={'ResponseTime': 'float32'}` вҶ’ 50 % smaller floats
- Incremental `dict` aggregation вҶ’ no growing DataFrame
- `del chunk` + `gc.collect()` вҶ’ immediate memory release
